# 开始使用 Mojo🔥
在此页面上，我们将向您展示如何使用 Mojo 创建经典的“Hello world”入门程序。如果您更愿意了解如何编写 Mojo 代码，请参阅 Mojo 简介。

安装 Mojo 即表示您了解并同意我们的软件许可。

## 1、创建新项目
要创建新的 Mojo 项目，我们将使用 Magic — 基于 conda 的虚拟环境管理器和包管理器。

  1.使用以下命令在 macOS 或 Ubuntu Linux 上安装 Magic：
  ```bash
  curl -ssL https://magic.modular.com | bash
  ```
  然后运行终端中打印的源命令。
  
  2.创建一个名为“hello-world”的Mojo项目：
  ```bash
  magic init hello-world --format mojoproject
  ```
  这将创建一个名为 hello-world 的目录并安装 Mojo 项目依赖项 - Mojo 项目的唯一依赖项是 max 包（Mojo 与 MAX 捆绑在一起）。

  3.在项目虚拟环境中启动一个shell：
  ```bash
  cd hello-world && magic shell
  ```
  就是这样！ magic shell 命令激活虚拟环境，因此您现在可以开始使用 Mojo。例如，您可以像这样检查您的 Mojo 版本：
  ```bash
  mojo --version
  ```

  <Callout>
  安装 Mojo 夜间版本:  
  要安装最新的夜间版本，请在初始化项目时指定 max-nightly 频道：

    ```sh
    magic init hello-world-nightly --format mojoproject \
      -c conda-forge -c https://conda.modular.com/max-nightly
    ```

    ```sh
    cd hello-world-nightly && magic shell
    ```
  </Callout>

  ## 2、在 REPL 中运行代码
  首先，让我们使用 Mojo [REPL](https://en.wikipedia.org/wiki/Read%E2%80%93eval%E2%80%93print_loop)，它允许您在命令提示符中编写和运行 Mojo 代码：

1. 要启动 REPL 会话，请键入`mojo`并按<kbd>Enter</kbd>。
    
2. 输入`print("Hello, world!")`并按<kbd>Enter</kbd>两次（需要一个空行来表示表达式的结束）。
    
    结果如下：
    
```mojo
  $ mojo
  Welcome to Mojo! 🔥

  Expressions are delimited by a blank line.
  Type `:quit` to exit the REPL and `:mojo help repl` for further assistance.  

    1> print("Hello world")  
  Hello world
```
    
3. 要退出 REPL，请键入`:quit`并按<kbd>Enter</kbd>，或者按<kbd>Ctrl</kbd>\+ <kbd>D</kbd>。
    

您可以在 REPL 中编写任意多的代码。您可以按<kbd>Enter</kbd>开始新行并继续编写代码，当您希望 Mojo 评估代码时，请按<kbd>Enter</kbd>两次。如果有要打印的内容，Mojo 会将其打印出来，然后将提示返回给您。

REPL 主要用于短时间的实验，因为代码不会被保存。所以当你想写一个真正的程序时，你需要在`.mojo`源文件中写代码。

## 3、运行 Mojo 文件
现在让我们在 Mojo 源文件中编写代码并使用以下 `mojo`命令运行它：

1. 创建一个名为`hello.mojo`（或`hello.🔥`）的文件并添加以下代码：
    
    ```
    fn main():    print("Hello, world!")
    ```
    
    这就是您所需要的。保存文件并返回到您的终端。
    
2. 现在使用以下命令运行它`mojo`：
    
    ```
    mojo hello.mojo
    ```
    
    ```
    Hello, world!
    ```

## 4、构建可执行二进制文件
最后，让我们构建并运行相同的代码作为可执行文件：

1. 使用以下命令创建可执行文件`build`：
    
    ```
    mojo build hello.mojo
    ```
    
    可执行文件使用与文件相同的名称`.mojo`，但您可以使用选项进行更改`-o`。
    
2. 然后运行可执行文件：
    
    ```
    ./hello
    ```
    
    ```
    Hello, world!
    ```
    

该`build`命令创建一个静态编译的二进制文件，因此它包含运行所需的所有代码和库。

您现在只需键入以下内容即可停用虚拟环境`exit`：

```
exit
```

现在让我们尝试运行现有的代码示例。


## 5、运行来自 GitHub 的示例
我们在 GitHub 上的 Mojo 代码示例包含一个 Magic 配置文件，因此您可以简单地克隆 repo 并使用 运行代码`magic`。例如：

1. 克隆 Mojo 仓库：
    
    ```
    git clone https://github.com/modularml/mojo.git
    ```
    
    仅当您安装了夜间版本时，才签出夜间分支：
    
    ```
    git checkout nightly
    ```
    
2. 导航至示例：
    
    ```
    cd mojo/examples
    ```
    
3. 运行一些代码：
    
    ```
    magic run mojo hello_interop.mojo
    ```
    
    ```
    Hello Mojo 🔥!963Hello from Python!I can even print a numpy array:  [1 2 3]
    ```

## 更新Mojo
要更新项目中的 Mojo 版本，请使用`magic add` 并指定`max`包版本。

例如，如果您想始终使用最新版本的 Mojo，您可以使用通配符`*`作为版本，然后简单运行`magic update`（必须`magic add`在项目路径内运行）：

```
cd hello-world
```

```
magic add max
```

```
magic update
```


虽然通配符选项允许`magic update`始终安装最新版本，但它也会`magic.lock`使用您安装的明确版本更新项目中的文件。这可确保初始化项目的任何其他人也获得相同的软件包版本（直到您 `magic update`再次运行）。

要更具体地说明软件包版本，可以使用任何[Python 软件包版本说明符](https://packaging.python.org/en/latest/specifications/version-specifiers/#id5)。例如：

```
magic add "max~=24.4"
```

```
magic add "max>=24.4,<24.5"
```
